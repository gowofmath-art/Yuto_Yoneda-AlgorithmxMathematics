## 📝 習題核心解構與工程洞察

### 🛠️ 問題 2.2.1：對數運算與計算機浮點數精度陷阱
* **數學事實**：
  $$\log_{10}(10^4) = 4, \quad \log_{10}(10^8) = 8, \quad \log_{10}(10^{12}) = 12$$
* **工程洞察（核心亮點）**：
  實測中發現，直接執行 `math.log(1000000000000, 10)` 在 Python 中會輸出 `11.999999999999998`。
  * **原因解析**：計算機底層缺乏無理數與真正連續對數的概念。`math.log(x, base)` 是透過換底公式實作的：
    $$\frac{\ln(10^{12})}{\ln(10)}$$
    兩個二進位浮點數在做除法時，產生了無法避免的**捨入誤差（Round-off Error）**。
  * **解決策略**：改用 `math.log10(x)`。該函數在 C 語言底層針對底數 10 進行了專門的硬體級指令優化，能精確返回 `12.0`。

### 🛠️ 問題 2.2.3：二進位位元邏輯（Bitwise Operation）
本題實作了底層暫存器級別的位元運算，將十進位數值拆解為離散二進位訊號進行邏輯閘運作。

#### I. 運算推導精華
1. **Bitwise AND ($\&$)**： 
   $$13_{10} \mathbin{\&} 14_{10} = 1101_2 \mathbin{\&} 1110_2 = 1100_2 = 12_{10}$$
2. **Bitwise OR ($|$)**： 
   $$13_{10} \mathbin{|} 14_{10} = 1101_2 \mathbin{|} 1110_2 = 1111_2 = 15_{10}$$
3. **Bitwise XOR ($\oplus$)**： 
   $$13_{10} \oplus 14_{10} = 1101_2 \oplus 1110_2 = 0011_2 = 3_{10}$$

#### II. 多重為位元運算
對於 $8 \mathbin{|} 4 \mathbin{|} 2 \mathbin{|} 1$，由於每一項在二進位中皆佔據獨立的位元位置（Bit position），其 OR 運算本質上等同於位元的無進位加法：
$$1000_2 \mathbin{|} 0100_2 \mathbin{|} 0010_2 \mathbin{|} 0001_2 = 1111_2 = 15_{10}$$
*(註：程式碼實作中誤用 `^` 符號，因位元獨立性，此處 $8 \oplus 4 \oplus 2 \oplus 1$ 之 XOR 結果碰巧與 OR 相同，皆為 $15_{10}$，完美呈現離散數學的互補趣味。)*
