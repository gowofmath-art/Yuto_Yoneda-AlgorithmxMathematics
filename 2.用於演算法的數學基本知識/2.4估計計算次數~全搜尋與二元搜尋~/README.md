## 為何時間複雜度重要？
就算電腦算很快，可是電腦計算的速度也是有極限的，一般電腦每秒進行約 10^9次計算，雖然很快，可是若遇到需要超大量計算的「效率差的演算法」也會花很多時間。

因此，寫出漂亮的演算法，降低時間複雜度將是重點。本章節將從最簡單的時間複雜度的理解，到真正的數學定義都好好說明清楚。


本章節最重要的：
## 大 $O$ 符號（Big O Notation）的嚴謹數學定義

設 $f(N)$ 與 $g(N)$ 為定義在正實數或正整數上的兩個函數。
若存在兩個正常數（Positive Constants）$c$ 與 $N_0$，使得：

$$\forall N \ge N_0, \exists c > 0, \quad 0 \le f(N) \le c \cdot g(N)$$

則我們可將此關係記作：

$$f(N) = O(g(N))$$

---

### 🔍 核心概念解析

1. **集合的概念（最嚴謹的視角）**：
   在高等數學中，$O(g(N))$ 其實是一個**函數集合**。
   $$O(g(N)) = \{ f(N) \mid \exists c > 0, N_0 > 0 \text{ 使得 } \forall N \ge N_0, 0 \le f(N) \le c \cdot g(N) \}$$
   因此，寫成 $f(N) = O(g(N))$ 是一種習慣用法，更嚴謹的數學表示法其實是 **$f(N) \in O(g(N))$**。

2. **漸近上界（Asymptotic Upper Bound）**：
   這個定義的意思是，當 $N$ 成長到足夠大（超過 $N_0$）之後，函數 $f(N)$ 的增長速度將會被常數倍的 $g(N)$ 壓在下方。大 $O$ 符號描述的是演算法在「最壞情況（Worst Case）」下的時間複雜度趨勢。


時間複雜度也是我很喜歡的一個概念。我們藉由討論這個，可以知道一個演算法到底是快還是慢。不用所謂的速率，而是用「大概算幾次」來呈現這樣的指標。是我踏入程式世界中初期最感興趣的一部分。
