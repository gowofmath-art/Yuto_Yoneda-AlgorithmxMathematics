本篇章研讀米田優俊《演算法×數學》第 3.6 節，本章節探討排序，也就是將數個值由小到大的順序來排列的操作，也是演算法的大重點。本章節將介紹最常見簡單的方法「選擇排序」、高效演算法著稱的「合併排序」。此外，也要熟悉在理解合併排序時很重要的「遞迴的思考方式」。  

本章節可謂演算法中非常重要的主題。在此詳細記錄。  

以下為各小節：  

## 3.6.1 排序是什麼？  
將數個資料(數值、文字)以特定的順序進行排列的操作，稱為排序。  

## 3.6.2 選擇排序  
選擇排序是反覆尋找尚未檢查過的數之中最小的數，來將陣列排序的方法，演算法的步驟如下所示意。  
反覆進行 $N - 1$ 次以下操作。在第 $i$ 次操作中，進行以下事項。  

1. 在未排序部分（從 $A_i$ 到 $A_N$）中尋找最小的元素 $A_{min}$  
2. 將 $A_i$ 和 $A_{min}$ 交換  

此時，操作後的 $A_1、A_2、\dots、A_N$ 依從小到大的順序排列。  

---  

### 複雜度分析  
* **比較與計算次數**：  
  $$(N - 1) + (N - 2) + \dots + 1 = \frac{N(N - 1)}{2}$$  
* **時間複雜度**：$O(N^2)$  


## 3.6.3 什麼是遞迴？  
在描述演算法時，引用自身的形式來定義稱為「遞迴定義」。呼叫自身的函式稱為「遞迴函式」  


## 3.6.4 遞迴定義的例子：求5的階乘  
試著不要使用for迴圈來處理，如何撰寫5的階乘的演算法？  
以下為遞迴函式的演算法步驟：  
**操作 $N$：**

* $N$ 為 1 時：回傳 1
* $N$ 為 2 以上時：回傳（操作 $N - 1$ 的計算結果）$\times N$

※ 在操作 $N$ 的計算結果中，引用了同種操作的操作 $N - 1$，因此可以說是遞迴的。

8. 由於 `func(3)` 的呼叫結束，`func(4)` 可以回傳值。回傳 $6 \times 4 = 24$
9. 由於 `func(4)` 的呼叫結束，`func(5)` 可以回傳值。回傳 $24 \times 5 = 120$

## 3.6.7 遞迴函式例(3)：以分割統治法求合計值
分割統治法：把大問題拆成小問題，再把答案組回來  

**分割統治法（Divide and Conquer）**是一種演算法設計思想：  
將一個規模較大的問題，分解成數個結構相同、規模較小的子問題；  分別解決子問題後，再合併成原問題的答案。  
它不是某一支特定演算法，而是一套「思考方式」。  

注意：分割統治不等於遞迴  
遞迴是一種程式實作方式：函式呼叫自己。
分割統治是一種演算法策略：將問題拆成數個較小且結構相同的問題。它可以用遞迴實作，也可能用迭代實作。

因此兩者的關係是：分割統治是思想，遞迴是常見工具。  

更應理解以下幾點：  
1. 問題如何切割？  
2. 遞迴何時停止？  
3. 子問題是否獨立？  
4. 如何合併答案？  
5. 切割是否平衡？  
6. 遞迴關係式怎麼寫？  
7. 遞迴樹每一層做多少工作？  

其中最關鍵的思維是：  
不要直接解決規模為 n 的問題，而是思考：假設規模更小的問題已經能解決，我要怎麼利用它解決現在的問題？

## 3.6.9 合併排序  
合併排序是一種高效的排序演算法，其核心思想是建立在「分治法 (Divide and Conquer)」與「遞迴」之上。

* **計算複雜度**：$O(N \log N)$
* **基本基礎**：將兩個「已經排序好」的陣列，融合成一個新的、依序排列的陣列（即 `Merge` 操作）。

---

### 1. 核心演算法步驟（分治法）
當我們要對一個包含 $k$ 個元素的陣列進行合併排序時：
1. **分割 (Divide)**：將陣列對半分割為陣列 $A$ 與陣列 $B$（各含約 $k/2$ 個元素）。
2. **遞迴排序 (Conquer)**：對陣列 $A$ 進行合併排序得到 $A'$；對陣列 $B$ 進行合併排序得到 $B'$。
3. **合併 (Combine)**：透過 `Merge` 操作，將已排序的 $A'$ 與 $B'$ 合併，產出最終排序好的陣列。

---

### 2. 關鍵底層：Merge（合併）操作
給定長度為 $a$ 的已排序陣列 $A$ 與長度為 $b$ 的已排序陣列 $B$，將其合併為新陣列 $C$ 的步驟如下（最多需要 $a + b - 1$ 次比較，時間複雜度為 $O(a + b)$）：

1. 準備一個空陣列 $C$。
2. 重複以下操作，直到 $A$ 和 $B$ 的所有元素都被取完：
   * 如果 **$A$ 為空**：將 $B$ 中剩下的最小元素（最前端）移至 $C$。
   * 如果 **$B$ 為空**：將 $A$ 中剩下的最小元素（最前端）移至 $C$。
   * 如果 **兩者皆非空**：比較 $A$ 與 $B$ 目前最前端的元素，將**較小者**移至 $C$。

---

### 3. 陣列追蹤範例

以陣列 `[31, 41, 59, 26]` 為例，其背後的完整拆解與合併過程如下：

#### 拆解階段（由上往下分割）
```text
      [31, 41, 59, 26]
         /        \
   [31, 41]      [59, 26]
   /      \      /      \
 [31]    [41]  [59]    [26]  <- 分割至最小單位（單一元素視為已排序）
```

### 合併階段（由下往上遞迴 Merge） 
```text
[31]    [41]  [59]    [26]
   \      /      \      /
   [31, 41]      [26, 59]    <- 第一輪 Merge：[59]與[26]比較後排序
       \            /
      [26, 31, 41, 59]       <- 第二輪 Merge：比較兩個已排序陣列的最前端
```

雙指標 Merge 過程微觀放大（最後一輪）：  
比較 $A'=[31, 41]$ 的 31 與 $B'=[26, 59]$ 的 26 $\rightarrow$ 26 較小 $\rightarrow$ $C=[26]$  
比較 $A'=[31, 41]$ 的 31 與 $B'=[59]$ 的 59 $\rightarrow$ 31 較小 $\rightarrow$ $C=[26, 31]$  
比較 $A'=[41]$ 的 41 與 $B'=[59]$ 的 59 $\rightarrow$ 41 較小 $\rightarrow$ $C=[26, 31, 41]$  
此時 $A'$ 已空，直接將 $B'$ 剩下的 59 移入 $\rightarrow$ 最終得到 $C=[26, 31, 41, 59]$  
---  

## 3.6.10 合併排序的計算複雜度  
### 為什麼合併排序的時間複雜度是 $O(N \log N)$？

合併排序的時間複雜度主要由兩個維度決定：
1. **樹的高度（層數）**：決定了我們需要「切幾次」與「合併幾輪」。
2. **每一層的代價（工作量）**：決定了每一輪合併時，電腦需要做多少次比較。

我們可以透過結構化的幾何圖形來推導：

### 1. 為什麼是 $\log N$？（縱向的層數）
當我們對一個長度為 $N$ 的陣列進行「對半分割」，每一次分割陣列長度就會除以 2。

假設我們分割了 $h$ 次才把陣列拆到只剩下單一元素（長度為 1）：
$$N \rightarrow \frac{N}{2} \rightarrow \frac{N}{4} \rightarrow \dots \rightarrow \frac{N}{2^h} = 1$$

由 $\frac{N}{2^h} = 1$ 可以推導出：
$$2^h = N \implies h = \log_2 N$$

這意味著，不論初始陣列的數字如何排列，將其完全拆解與合併的**總層數固定就是 $\log_2 N$ 層**。

---

### 2. 為什麼每層的代價是 $O(N)$？（橫向的工作量）
我們來觀察「由下往上」每一層進行 `Merge` 操作時的總工作量：

```text
第 0 層 (頂層):   [             N 筆資料             ]  -> 合併總工作量 = N
                      /                       \
第 1 層:       [   N/2 筆   ]             [   N/2 筆   ]  -> 合併總工作量 = 2 * (N/2) = N
                /         \                 /         \
第 2 層:     [N/4]       [N/4]           [N/4]       [N/4]  -> 合併總工作量 = 4 * (N/4) = N
```

## 3.6.11 其他的排序演算法  
最後提及幾個代表性的排序演算法：
1.  插入排序：從前面開始按照順序反覆將元素適當地插入到已完成排序部分的排序法。若在幾乎以小到大排序好的情況下，會很快，但若沒有，時間複雜度最差會是 $O(n^2)$  
2.  快速排序：與合併排序法類似，使用分割統治法的想法，時間複雜度為 $O(nlog(n))$ 是常見且高效的通用排序演算法之一。  
3.  計數排序：計算陣列中不同數值出現的個數(針對資料的特性先做統計)，在進行排序的特殊排序演算法，時間複雜度為 $O(n+k)$，k為不同種個數。  

比較：

| 演算法  |           最佳 |           平均 |           最差 |   額外空間 |
| ---- | -----------: | -----------: | -----------: | -----: |
| 選擇排序 |     $O(N^2)$ |     $O(N^2)$ |     $O(N^2)$ | $O(1)$ |
| 合併排序 | $O(N\log N)$ | $O(N\log N)$ | $O(N\log N)$ | $O(N)$ |
| 快速排序 | $O(N\log N)$ | $O(N\log N)$ |     $O(N^2)$ |  視實作而定 |
| 計數排序 |     $O(N+K)$ |     $O(N+K)$ |     $O(N+K)$ | $O(K)$ |
